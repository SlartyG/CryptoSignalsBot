# MVP-сигналы

Документ описывает первую версию сигналов для быстрой реализации. Все сигналы строятся на публичных данных Bybit V5 API — ключ не требуется.

## Принципы MVP

- **Не «купи/продай»**, а алерты о рыночных аномалиях с краткой интерпретацией.
- **Редкие и понятные** — лучше 5–15 качественных алертов в день по 10 парам, чем сотни шумных.
- **Cooldown** — не повторять один и тот же тип сигнала по паре чаще, чем раз в 30–60 минут.
- **Единый формат сообщения** — пара, тип, направление, цифры, уровень уверенности, disclaimer.

### Формат алерта в Telegram

```
🔔 BTCUSDT · Funding Extreme
📊 Funding: +0.082% (8h) · avg 24h: +0.021%
📈 Направление: bearish bias (перегретые лонги)
⚡ Уверенность: medium
ℹ️ Не финансовая рекомендация
```

## Список рынков

Топ-10 linear USDT-пар по `turnover24h` из `/v5/market/tickers?category=linear`.

- Обновлять список **раз в сутки** (00:00 UTC).
- Всегда включать BTCUSDT и ETHUSDT, даже если временно выпали из топ-10.
- **Free-тариф:** только BTCUSDT.
- **Paid-тариф:** все 10 пар, высокий приоритет в очереди рассылки.

## Сигнал 1: Экстремальный Funding Rate

### Зачем трейдеру

Funding показывает перекос long/short на перпах. Экстремально положительный funding — лонги платят шортам (толпа в лонгах, риск отката). Экстремально отрицательный — наоборот (риск short squeeze).

### Источник данных

| Метод | Endpoint |
|-------|----------|
| REST | `GET /v5/market/funding/history` |
| Параметры | `category=linear`, `symbol`, `limit=48` |

Polling: каждые **5 минут**.

### Условия срабатывания

| Уровень | Условие | Интерпретация |
|---------|---------|---------------|
| Medium | \|funding\| ≥ 0.05% за период | Перекос позиций |
| High | \|funding\| ≥ 0.10% за период | Сильный перекос, повышенный риск flush/squeeze |

Дополнительно (усиление до High): funding вырос более чем в **2×** от среднего за последние 24h.

### Cooldown

60 минут на пару.

### Реализация

1. Запросить последний funding для каждой пары из списка.
2. Посчитать среднее за 24h (3 периода по 8h или все доступные точки).
3. Сравнить с порогами, отправить алерт.

## Сигнал 2: Open Interest + движение цены

### Зачем трейдеру

OI — сумма открытых деривативных позиций. Рост OI вместе с ценой = новый капитал в тренде. Рост OI при падении = новые шорты. Падение OI при резком движении = ликвидации, а не новые позиции.

### Источник данных

| Метод | Endpoint |
|-------|----------|
| REST OI | `GET /v5/market/open-interest` — `intervalTime=5min`, `limit=12` |
| REST цена | `GET /v5/market/kline` — `interval=60`, `limit=4` |

Polling: каждые **5 минут**.

### Условия срабатывания

Изменение OI за **1 час** (12 × 5min):

| Паттерн | OI | Цена (1h) | Сигнал |
|---------|-----|-----------|--------|
| Trend long | +8% и более | +2% и более | Bullish — новые лонги |
| Trend short | +8% и более | −2% и более | Bearish — новые шорты |
| Liquidation flush | −10% и более | \|Δ\| ≥ 3% | Neutral — каскад ликвидаций |

Пороги для альтов (не BTC/ETH): OI ±12%, цена ±3%.

### Cooldown

45 минут на пару и тип паттерна.

### Реализация

1. Взять OI `t0` и `t-1h`, посчитать `% change`.
2. Взять close свечи 1h назад и текущую цену из ticker.
3. Классифицировать паттерн, отправить алерт.

## Сигнал 3: Всплеск ликвидаций

### Зачем трейдеру

Крупные каскады ликвидаций усиливают движение цены и часто отмечают локальные экстремумы или продолжение импульса.

### Источник данных

| Метод | WebSocket topic |
|-------|-----------------|
| WS | `allLiquidation.{symbol}` (public, linear) |

Push frequency: 500ms. Один WS-подключение на все пары из списка.

### Условия срабатывания

Агрегировать ликвидации в скользящем окне **15 минут**:

| Уровень | Условие |
|---------|---------|
| Medium | Сумма ликвидаций ≥ **$500K** USD (size × price) |
| High | Сумма ≥ **$2M** USD **или** ≥ 3× среднего за последние 24h для этой пары |

Отдельно указывать доминирующую сторону: `Buy` в WS = ликвидированы **лонги**, `Sell` = **шорты**.

### Cooldown

30 минут на пару.

### Реализация

1. Подписаться на `allLiquidation.*` для топ-10.
2. Буфер событий в Redis/in-memory с TTL 15 min.
3. Конвертировать `v` (size) × `p` (price) в USD.
4. При превышении порога — алерт с breakdown long/short liquidations.

## Сигнал 4: Volume Spike

### Зачем трейдеру

Аномальный объём часто предшествует пробою диапазона или усилению тренда.

### Источник данных

| Метод | Endpoint |
|-------|----------|
| REST | `GET /v5/market/kline` — `interval=60`, `limit=168` (7 дней) |

Polling: каждые **15 минут** (на закрытии часовой свечи + offset 1 min).

### Условия срабатывания

- Объём **текущей часовой свечи** (или только что закрытой) ≥ **2.5×** медианы объёма за те же часы суток за последние 7 дней.
- Для High: ≥ **4×** медианы **и** \|изменение цены за час\| ≥ 1.5%.

### Cooldown

60 минут на пару.

### Реализация

1. Собрать 168 hourly volumes, сгруппировать по часу UTC (0–23).
2. Медиана для текущего часа.
3. Сравнить с объёмом последней свечи.

## Сводная таблица MVP

| # | Сигнал | Источник | Polling / Stream | Сложность |
|---|--------|----------|------------------|-----------|
| 1 | Funding Extreme | REST funding/history | 5 min | Низкая |
| 2 | OI + Price | REST open-interest + kline | 5 min | Средняя |
| 3 | Liquidation Spike | WebSocket allLiquidation | Real-time | Средняя |

**Оценка сроков:** 1–2 недели на collector + scorer + Telegram notifier для всех четырёх типов.

---

## Архитектура (минимальная)

```
Bybit API ──► Collectors ──► Signal Engine ──► Queue ──► Telegram Bot
                  │                │
                  │                └── cooldown, dedup, scoring
                  └── REST pollers + WS liquidation listener
```

### Компоненты

| Комponent | Назначение |
|-----------|------------|
| `market_universe` | Список топ-10 пар, обновление раз в сутки |
| `funding_collector` | Polling funding rate |
| `oi_collector` | Polling OI + price |
| `liquidation_listener` | WS-подписка, агрегация |
| `volume_collector` | Polling klines |
| `signal_engine` | Правила, пороги, cooldown |
| `notifier` | Форматирование и отправка в TG с учётом тарифа |

### Хранение состояния

- **Cooldown keys:** `{symbol}:{signal_type}` → timestamp последнего алерта.
- **Liquidation buffer:** sliding window 15 min per symbol.
- **24h averages:** funding avg, liquidation avg — обновлять инкрементально.

## Пороги по умолчанию (конфиг)

Все пороги вынести в конфиг/env для A/B и админ-подстройки без деплоя:

```yaml
funding:
  medium_threshold: 0.0005   # 0.05%
  high_threshold: 0.001      # 0.10%
  spike_multiplier: 2.0

oi:
  change_1h_pct: 8
  price_change_1h_pct: 2
  liquidation_oi_drop_pct: 10

liquidation:
  window_minutes: 15
  medium_usd: 500000
  high_usd: 2000000

volume:
  spike_multiplier: 2.5
  high_spike_multiplier: 4.0
  min_price_change_pct: 1.5

cooldown_minutes:
  funding: 60
  oi: 45
  liquidation: 30
  volume: 60
```

## Что не входит в MVP

- On-chain whale alerts (Whale Alert, Glassnode) — отдельный источник, платные API.
- Confluence score (комбинация нескольких сигналов) — v2.
- TA-пакет (RSI, EMA cross) — v2.
- Order book imbalance — v2.
- Cross-exchange данные — v2.
